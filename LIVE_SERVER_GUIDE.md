# Инструкция по запуску через Live Server

## Быстрый старт

1. **Откройте файл `webapp/index.html` в VS Code**
   - Это веб-интерфейс приложения с поддержкой демо-режима

2. **Запустите Live Server одним из способов:**
   - **Способ 1:** Правой кнопкой мыши на `webapp/index.html` → "Open with Live Server"
   - **Способ 2:** Нажмите кнопку "Go Live" в правом нижнем углу VS Code (в статус-баре)
   - **Способ 3:** Используйте горячую клавишу `Alt + L, Alt + O` (или настройте свою)

3. **Браузер откроется автоматически** с адресом типа `http://127.0.0.1:5500/webapp/index.html`

## Важные замечания

- **Демо-режим:** Приложение будет работать в демо-режиме (без подключения к API серверу)
  - Вы сможете тапать по монете и видеть анимации
  - Данные будут сохраняться только локально в браузере
  - Рефералы и лидерборд не будут работать без API сервера

- **Для полной функциональности:** Если хотите подключить к API серверу (порт 8080), запустите `main.py` отдельно:
  ```bash
  python main.py
  ```

## Альтернативные варианты

### Вариант 1: Корневой `index.html`
Можно также открыть `index.html` в корне проекта, но он требует настройки API сервера.

### Вариант 2: React версия
Для запуска React версии (`webapp-react/index.html`):
1. Убедитесь, что установлены зависимости (если нужно)
2. Откройте `webapp-react/index.html` через Live Server

## Структура файлов

```
webapp/
├── index.html    ← Откройте этот файл через Live Server
├── script.js     ← JavaScript логика (с демо-режимом)
└── style.css     ← Стили приложения
```

**Важно:** Корневые файлы `index.html` и `script.js` были удалены, так как они дублировали функциональность и имели ошибки. Используйте только версию из папки `webapp/`.

## Решение проблем

- **Стили не загружаются?** Проверьте, что файлы `style.css` и `script.js` находятся в той же папке, что и `index.html`
- **Ошибки в консоли?** Откройте DevTools (F12) и проверьте ошибки
- **Live Server не запускается?** Убедитесь, что расширение "Live Server" установлено и активировано
